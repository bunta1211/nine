# バックアップ・リカバリ戦略

## 1. 概要

本ドキュメントは、システム障害やバグ発生時に「2日前の状態に戻す」ための
バックアップ・リカバリ手順を定義する。

### 1.1 設計方針

| 項目 | 方針 |
|------|------|
| 対象者 | プログラミング素人でも実行可能 |
| 目標復旧時間 | 2時間以内 |
| バックアップ保持期間 | 7日間 |
| バックアップ頻度 | 日次（毎日深夜3時） |

---

## 2. バックアップ対象

### 2.1 バックアップが必要なもの

| 対象 | 場所 | 重要度 | バックアップ方法 |
|------|------|:------:|------------------|
| **データベース** | MySQL (social9) | ★★★ | mysqldump |
| **アップロードファイル** | /nine/uploads/ | ★★★ | ファイルコピー |
| **ソースコード** | /nine/ | ★★☆ | Git |
| **設定ファイル** | /nine/config/ | ★★★ | ファイルコピー |
| **ログファイル** | /nine/logs/ | ★☆☆ | 自動保存 |

### 2.2 バックアップ不要なもの

| 対象 | 理由 |
|------|------|
| node_modules | npm installで復元可能 |
| vendor | composer installで復元可能 |
| キャッシュファイル | 自動生成される |

```

├── daily\                    ← 日次バックアップ（7日分）
│   ├── 2024-12-24\

│   │   ├── uploads\
```

| GitHub（private） | コードのみ | git push |

## 4. 自動バックアップ設定

### 4.1 バックアップスクリプト

以下のファイルを作成して、Windowsタスクスケジューラで毎日実行する。

**ファイル: `C:\xampp\backup\scripts\daily_backup.bat`**

```batch
@echo off
REM ============================================
REM 日次バックアップスクリプト
REM 毎日深夜3時に実行
REM ============================================

REM 日付取得
for /f "tokens=1-3 delims=/" %%a in ('date /t') do set TODAY=%%a%%b%%c
set TODAY=%date:~0,4%%date:~5,2%%date:~8,2%

REM バックアップ先ディレクトリ
set BACKUP_DIR=C:\xampp\backup\daily\%TODAY%
set DB_NAME=social9
set MYSQL_USER=root
set MYSQL_PASS=

REM ディレクトリ作成
if not exist "%BACKUP_DIR%\database" mkdir "%BACKUP_DIR%\database"
if not exist "%BACKUP_DIR%\uploads" mkdir "%BACKUP_DIR%\uploads"
if not exist "%BACKUP_DIR%\config" mkdir "%BACKUP_DIR%\config"

REM 1. データベースバックアップ
echo [%date% %time%] データベースバックアップ開始...
"C:\xampp\mysql\bin\mysqldump.exe" -u %MYSQL_USER% %DB_NAME% > "%BACKUP_DIR%\database\%DB_NAME%_%TODAY%.sql"
echo [%date% %time%] データベースバックアップ完了

REM 2. アップロードファイルバックアップ
echo [%date% %time%] アップロードファイルバックアップ開始...
"C:\Program Files\7-Zip\7z.exe" a -tzip "%BACKUP_DIR%\uploads\uploads_%TODAY%.zip" "C:\xampp\htdocs\nine\uploads\*" -r
echo [%date% %time%] アップロードファイルバックアップ完了

REM 3. 設定ファイルバックアップ
echo [%date% %time%] 設定ファイルバックアップ開始...
"C:\Program Files\7-Zip\7z.exe" a -tzip "%BACKUP_DIR%\config\config_%TODAY%.zip" "C:\xampp\htdocs\nine\config\*"
echo [%date% %time%] 設定ファイルバックアップ完了

REM 4. 7日以上前のバックアップを削除
echo [%date% %time%] 古いバックアップを削除中...
forfiles /p "C:\xampp\backup\daily" /d -7 /c "cmd /c if @isdir==TRUE rmdir /s /q @path" 2>nul

REM 5. 完了ログ
echo [%date% %time%] バックアップ完了: %BACKUP_DIR% >> C:\xampp\backup\backup.log

echo ============================================
echo バックアップが完了しました
echo 保存先: %BACKUP_DIR%
echo ============================================
pause
```

### 4.2 タスクスケジューラ設定手順

1. Windowsキー + R → `taskschd.msc` を入力してEnter
2. 「タスクの作成」をクリック
3. 以下を設定：

| タブ | 項目 | 設定値 |
|------|------|--------|
| 全般 | 名前 | Social9 日次バックアップ |
| 全般 | 最上位の特権で実行 | ✓ チェック |
| トリガー | 開始 | 毎日 3:00 |
| 操作 | プログラム | C:\xampp\backup\scripts\daily_backup.bat |

## 5. 手動バックアップ手順

### 5.1 重要な変更前のバックアップ

大きな変更（機能追加、DB変更等）の前に必ず実行。

#### データベースのみ（1分）

```
1. phpMyAdminを開く: http://localhost/phpmyadmin
2. 左側で「social9」を選択
3. 上部メニュー「エクスポート」をクリック
4. 「実行」ボタンをクリック
5. ダウンロードされたSQLファイルを保存
   → C:\xampp\backup\manual\before_変更内容_日付.sql
```

#### フルバックアップ（5分）

```
1. C:\xampp\backup\scripts\daily_backup.bat をダブルクリック
2. 完了メッセージを確認
3. 保存先フォルダを確認
```

## 6. リカバリ（復旧）手順

### 6.1 データベースの復旧

**所要時間: 5-30分（データ量による）**

#### 方法A: phpMyAdminを使用（推奨・簡単）

```
1. phpMyAdminを開く: http://localhost/phpmyadmin
2. 左側で「social9」を選択
3. 上部メニュー「インポート」をクリック
4. 「ファイルを選択」で復元したいSQLファイルを選択
   例: C:\xampp\backup\daily\2024-12-22\database\social9_20241222.sql
5. 「実行」ボタンをクリック
6. 「インポートは正常に終了しました」を確認
```

#### 方法B: コマンドライン（高速・大容量向け）

```batch
REM コマンドプロンプトを管理者で開く
cd C:\xampp\mysql\bin

REM データベースを復元（パスワードなしの場合）
mysql -u root social9 < "C:\xampp\backup\daily\2024-12-22\database\social9_20241222.sql"

REM パスワードありの場合
mysql -u root -p social9 < "C:\xampp\backup\daily\2024-12-22\database\social9_20241222.sql"
```

### 6.2 アップロードファイルの復旧

**所要時間: 5-15分**

```
1. 現在のuploadsフォルダをリネーム
   C:\xampp\htdocs\nine\uploads → uploads_broken

2. バックアップを解凍
   C:\xampp\backup\daily\2024-12-22\uploads\uploads_20241222.zip
   → C:\xampp\htdocs\nine\uploads\ に展開

3. 動作確認後、uploads_brokenを削除
```

### 6.3 ソースコードの復旧（Git使用）

**所要時間: 1-5分**

```bash
# 現在の変更を退避
git stash

# 2日前のコミットに戻す
git log --oneline  # コミット履歴を確認
git checkout <コミットID>  # 戻りたいコミットIDを指定

# または、特定のファイルだけ戻す
git checkout <コミットID> -- path/to/file.php
```

### 6.4 完全復旧チェックリスト

復旧後、以下を確認：

- [ ] ログインできるか
- [ ] メンバー一覧が表示されるか
- [ ] グループ一覧が表示されるか
- [ ] チャットが送受信できるか
- [ ] ファイルアップロードができるか
- [ ] 画像が表示されるか

## 7. ログの保存

### 7.1 ログファイルの場所

| ログ種別 | 場所 | 内容 |
|----------|------|------|
| PHPエラー | C:\xampp\php\logs\php_error_log | PHPのエラー |
| Apache | C:\xampp\apache\logs\error.log | Webサーバーエラー |
| MySQL | C:\xampp\mysql\data\mysql_error.log | DBエラー |
| **アプリログ** | C:\xampp\htdocs\nine\logs\ | 自作ログ |

### 7.2 アプリケーションログの設定

**ファイル: `C:\xampp\htdocs\nine\logs\` フォルダを作成**

```
C:\xampp\htdocs\nine\logs\
├── app_2024-12-24.log       ← アプリケーションログ（日次）
├── error_2024-12-24.log     ← エラーログ（日次）
├── audit_2024-12-24.log     ← 監査ログ（日次）
└── .htaccess                ← アクセス禁止設定
```

**.htaccessの内容:**
```apache
# ログファイルへの直接アクセスを禁止
Deny from all
```

### 7.3 ログ出力関数

**ファイル: `includes/logger.php`**

```php
<?php
/**
 * シンプルなログ出力関数
 */

define('LOG_DIR', __DIR__ . '/../logs/');

/**
 * ログを出力
 * @param string $message メッセージ
 * @param string $level レベル（info/warning/error）
 * @param string $type ログ種別（app/error/audit）
 */
function writeLog($message, $level = 'info', $type = 'app') {
    $date = date('Y-m-d');
    $time = date('Y-m-d H:i:s');
    $file = LOG_DIR . "{$type}_{$date}.log";
    
    // ログディレクトリがなければ作成
    if (!is_dir(LOG_DIR)) {
        mkdir(LOG_DIR, 0755, true);
    }
    
    $logLine = "[{$time}] [{$level}] {$message}" . PHP_EOL;
    file_put_contents($file, $logLine, FILE_APPEND | LOCK_EX);
}

/**
 * エラーログ
 */
function logError($message) {
    writeLog($message, 'error', 'error');
}

/**
 * 監査ログ（誰が何をしたか）
 */
function logAudit($userId, $action, $details = '') {
    $message = "user_id={$userId} action={$action} {$details}";
    writeLog($message, 'info', 'audit');
}
```

## 8. 緊急時対応フロー

### 8.1 障害発生時のフロー

```
┌─────────────────────────────────────────┐
│           障害発生！                     │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│ Step 1: 状況確認（5分）                  │
│ ・エラーメッセージを記録（スクショ）      │
│ ・いつから発生したか確認                 │
│ ・影響範囲を確認                         │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│ Step 2: 一時対応（5分）                  │
│ ・必要ならメンテナンスモードにする        │
│ ・ユーザーに告知                         │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│ Step 3: 原因特定（15-30分）              │
│ ・エラーログを確認                       │
│ ・最後に変更した内容を確認               │
│ ・AIに相談                              │
└─────────────────────────────────────────┘
                    ↓
        ┌─────────┴─────────┐
        ↓                   ↓
┌───────────────┐   ┌───────────────┐
│ 修正可能      │   │ 修正困難      │
│ → 修正して    │   │ → バックアップ │
│   デプロイ    │   │   から復旧    │
└───────────────┘   └───────────────┘
                    ↓
┌─────────────────────────────────────────┐
│ Step 4: 復旧確認                         │
│ ・チェックリストで動作確認               │
│ ・ユーザーに復旧告知                     │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│ Step 5: 振り返り                         │
│ ・原因と対策を記録                       │
│ ・再発防止策を検討                       │
└─────────────────────────────────────────┘
```

### 8.2 メンテナンスモード

障害対応中にユーザーアクセスを止める方法。

**ファイル: `C:\xampp\htdocs\nine\maintenance.html`**

```html
<!DOCTYPE html>
<html lang="ja">
<head>
    <meta charset="UTF-8">
    <title>メンテナンス中</title>
    <style>
        body { font-family: sans-serif; text-align: center; padding: 50px; background: #f5f5f5; }
        .container { background: #fff; padding: 40px; border-radius: 10px; max-width: 500px; margin: 0 auto; }
        h1 { color: #e74c3c; }
    </style>
</head>
<body>
    <div class="container">
        <h1>🔧 メンテナンス中</h1>
        <p>現在システムメンテナンスを行っています。</p>
        <p>しばらくお待ちください。</p>
        <p><small>復旧予定: ○○時頃</small></p>
    </div>
</body>
</html>
```

**メンテナンスモードON:**
```
index.php を index.php.bak にリネーム
maintenance.html を index.php にリネーム
```

**メンテナンスモードOFF:**
```
index.php を maintenance.html に戻す
index.php.bak を index.php に戻す
```
