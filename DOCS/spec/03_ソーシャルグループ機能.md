# ソーシャルグループ機能 (Social Groups)

## 概要

**ソーシャルグループ**とは、家族、老人会、子ども会、青年会議所、株式会社等の**まとまった集団**のこと。
法人⇔職員、家庭⇔児童の関係を**統一された「ソーシャルグループ」構造**で管理します。
家庭も一つのソーシャルグループとして扱うことで、シンプルなテーブル設計を実現します。

### ソーシャルグループアカウント

ソーシャルグループ（特に法人）は、個人アカウントと同様に**ソーシャルグループアカウント**を持つことができます。
メンバーには**肩書**（窓口、代表、事務局 等）を設定でき、対外的な表示に使用します。

---

## 🏢 ソーシャルグループの種類

| 種類 | 説明 | 例 |
|------|------|-----|
| **corporation** | 法人・企業 | 株式会社〇〇、青年会議所 |
| **family** | 家庭 | 田中家 |
| **school** | 学校・教育機関 | 〇〇インターナショナルスクール |
| **group** | 一般グループ | 老人会、子ども会、趣味サークル |

---

## 🏷️ 肩書機能

ソーシャルグループのメンバーには**肩書（title）**を設定できます。

| 肩書の例 | 用途 |
|----------|------|
| **代表** | グループの代表者として対外発信 |
| **窓口** | 問い合わせ対応の担当者 |
| **事務局** | 事務処理担当 |
| **広報** | 広報・PR担当 |
| **会長** | 老人会・子ども会等の会長 |
| **副会長** | 副会長 |

### 肩書の表示例

```
┌─────────────────────────────────────────────────────┐
│ 🏢 株式会社クローバー                               │
├─────────────────────────────────────────────────────┤
│                                                     │
│ 👤 山田太郎 【代表】 (owner)                        │
│ 👤 鈴木一郎 【窓口】 (admin)                        │
│ 👤 佐藤花子 【事務局】 (admin)                      │
│ 👤 田中次郎 (member)                                │
│                                                     │
└─────────────────────────────────────────────────────┘
```

---

## 👥 メンバーの役割

| 役割 | 権限 | 法人での例 | 家庭での例 |
|------|------|-----------|-----------|
| **owner** | 全権限 | 代表者 | 世帯主 |
| **admin** | 管理権限 | 管理職 | 保護者 |
| **member** | 一般権限 | 社員 | 成人家族 |
| **restricted** | 制限付き | インターン | 児童 |

---

| 操作 | owner | admin | member | restricted |
|------|:-----:|:-----:|:------:|:----------:|
| ソーシャルグループ設定変更 | ✅ | ❌ | ❌ | ❌ |
| メンバー追加/削除 | ✅ | ✅ | ❌ | ❌ |
| 役割・肩書変更 | ✅ | ✅ | ❌ | ❌ |
| 制限設定 | ✅ | ✅ | ❌ | ❌ |
| グループ作成 | ✅ | ✅ | ✅ | ⚠️ |
| 外部連絡許可 | ✅ | ✅ | ❌ | ❌ |
| 通話発信 | ✅ | ✅ | ✅ | ⚠️ |
| メッセージ送信 | ✅ | ✅ | ✅ | ✅ |

⚠️ = 設定により許可/禁止

---

## 👶 制限付きメンバー（restricted）の設定

未成年や制限が必要なメンバーに対する設定。

### 通信制限

| 設定項目 | 選択肢 |
|----------|--------|
| **外部通信** | 禁止 / グループ内のみ / 許可リストのみ / 通知のみ / 許可 |
| **通話制限** | 禁止 / グループのみ / 許可リストのみ / 許可 |
| **ファイル送信** | 禁止 / 許可 |

### 利用時間制限

| 設定項目 | 例 |
|----------|-----|
| 利用開始時間 | 07:00 |
| 利用終了時間 | 21:00 |
| 1日の利用上限 | 120分 |

### 監視設定

| 設定項目 | 説明 |
|----------|------|
| メッセージ閲覧 | 管理者がメッセージを見れるか |
| オンライン状態確認 | 管理者がオンライン状態を見れるか |
| 友達一覧確認 | 管理者が友達一覧を見れるか |

## 🏠 家庭（family）の運用例

### 登録フロー

```
1. 保護者がアカウント作成
2. 「ソーシャルグループを作成」で家庭作成（type: family）
3. 保護者は owner または admin として登録
4. 子どものアカウントを招待または代理作成
5. 子どもは restricted として参加
6. SNS設定を提案 → 子どもが同意
```

## 🏢 法人（corporation）の運用例

### 登録フロー

```

2. 「ソーシャルグループを作成」で法人作成（type: corporation）
```

## 🗄️ データベース設計

> **注意**: テーブル名は既存システムとの互換性のため `organizations`, `organization_members` を維持。
> アプリ内表示では「ソーシャルグループ」と呼称する。

### organizations テーブル（ソーシャルグループ）

```sql
CREATE TABLE organizations (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    
    name VARCHAR(100) NOT NULL,
    type ENUM('corporation', 'family', 'school', 'group') NOT NULL,
    description TEXT,
    icon_path VARCHAR(500),
    
    -- 設定
    default_member_role ENUM('member', 'restricted') DEFAULT 'member',
    require_admin_approval TINYINT(1) DEFAULT 0,
    
    created_by INT UNSIGNED NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    FOREIGN KEY (created_by) REFERENCES users(id),
    INDEX idx_type (type)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### organization_members テーブル（ソーシャルグループメンバー）

```sql
CREATE TABLE organization_members (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    organization_id INT UNSIGNED NOT NULL,
    user_id INT UNSIGNED NOT NULL,
    
    role ENUM('owner', 'admin', 'member', 'restricted') DEFAULT 'member',
    
    -- 肩書（窓口、代表、事務局 等）
    title VARCHAR(50) DEFAULT NULL,
    
    -- 制限設定（restricted用）
    external_contact ENUM('none', 'org_only', 'approved', 'notify', 'allow') DEFAULT 'allow',
    call_restriction ENUM('none', 'group_only', 'approved', 'allow') DEFAULT 'allow',
    file_send_allowed TINYINT(1) DEFAULT 1,
    
    -- 利用時間制限
    usage_start_time TIME,
    usage_end_time TIME,
    daily_limit_minutes INT UNSIGNED,
    
    -- 監視設定
    can_view_messages TINYINT(1) DEFAULT 0,
    can_view_online TINYINT(1) DEFAULT 1,
    can_view_contacts TINYINT(1) DEFAULT 1,
    
    joined_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    left_at DATETIME,
    
    FOREIGN KEY (organization_id) REFERENCES organizations(id) ON DELETE CASCADE,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    UNIQUE KEY unique_member (organization_id, user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### approved_contacts テーブル（許可済み連絡先）

```sql
CREATE TABLE approved_contacts (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    
    -- 制限対象のメンバー
    member_id INT UNSIGNED NOT NULL,
    
    -- 許可された連絡先
    approved_user_id INT UNSIGNED NOT NULL,
    
    -- 許可内容
    allow_dm TINYINT(1) DEFAULT 1,
    allow_call TINYINT(1) DEFAULT 1,
    
    -- 承認者
    approved_by INT UNSIGNED NOT NULL,
    approved_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    
    FOREIGN KEY (member_id) REFERENCES organization_members(id) ON DELETE CASCADE,
    FOREIGN KEY (approved_user_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (approved_by) REFERENCES users(id),
    UNIQUE KEY unique_approval (member_id, approved_user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## 🔗 API

| エンドポイント | メソッド | 機能 |
|---------------|:--------:|------|
| `/api/socialgroups.php?action=list` | GET | 所属ソーシャルグループ一覧 |
| `/api/socialgroups.php?action=get` | GET | ソーシャルグループ情報取得 |
| `/api/socialgroups.php?action=create` | POST | ソーシャルグループ作成 |
| `/api/socialgroups.php?action=update` | POST | ソーシャルグループ更新 |
| `/api/socialgroups.php?action=add_member` | POST | メンバー追加 |
| `/api/socialgroups.php?action=update_member` | POST | メンバー設定・肩書変更 |
| `/api/socialgroups.php?action=remove_member` | POST | メンバー削除 |
| `/api/socialgroups.php?action=approve_contact` | POST | 連絡先許可 |
| `/api/socialgroups.php?action=revoke_contact` | POST | 連絡先許可取り消し |

### 肩書設定の例

```json
{
  "action": "update_member",
  "organization_id": 1,
  "user_id": 5,
  "title": "窓口"
}
```
