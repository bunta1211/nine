# Social9 仕様書

## 概要

Social9は「すべての人々（1〜9）」のためのチャットアプリです。
子どもから大人まで安心して使え、地域の需要と供給をつなぐプラットフォームを目指します。

---

## 🏢 運営情報

| 項目 | 内容 |
|------|------|
| **運営会社** | 株式会社たげんご / 株式会社学童クローバークラブ |
| **発祥の地** | 愛知県岡崎市 |
| **初期展開** | クローバーインターナショナルスクール、学童クローバークラブ 等 |

---

## 🎯 6つの核心機能

| No. | 機能 | 概要 |
|:---:|------|------|
| 1 | **チャット** | テキスト・画像・ファイルの送受信 |
| 2 | **通話** | 音声通話・ビデオ通話（Jitsi Meet） |
| 3 | **ソーシャルグループ** | 法人・家庭・老人会等の集団を統一管理 |
| 4 | **翻訳** | 日本語・英語・中国語の自動翻訳 |
| 5 | **デザイン** | テーマ変更・スケルトンテーマ |
| 6 | **ユニバーサルマッチング** | 需要と供給を結びつける（収益の核） |

> **ソーシャルグループ (Social Groups)**: 家族、老人会、子ども会、青年会議所、株式会社等のまとまった集団
> ソーシャルグループには「窓口」「代表」「事務局」等の肩書を設定可能

## 🏗️ 技術構成

```
サーバー: heteml（本番）/ XAMPP（開発）
バックエンド: PHP 8.x（フレームワークなし）
データベース: MySQL 5.7+
フロントエンド: HTML/CSS/JavaScript（Vanilla）
通話: Jitsi Meet External API
言語: 日本語、英語、中国語
```

> **将来**: 1,000万人以上の利用者を想定する場合、AWS移転を検討。詳細は [AWS移転計画書](../AWS_MIGRATION_PLAN.md) を参照。

---

## 📐 設計原則

### 1. シンプルな構造
- フレームワーク不使用
- 1ファイル = 1機能
- ネストを浅く

### 2. 低負荷
- 必要な時だけDB接続
- シンプルなクエリ

### 3. 復旧が簡単
- FTPとphpMyAdminだけで復旧可能
- SQLファイルでスキーマ管理
