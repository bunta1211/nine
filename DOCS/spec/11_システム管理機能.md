# システム管理機能 仕様書

## 1. 概要

本機能は、チャットアプリケーション全体を統括管理するためのシステム開発者向け管理機能である。
複数の組織（法人・ファミリー等）を横断的に管理し、問題ユーザーへの対応を行う。

### 1.1 設計目標

| 項目 | 目標値 |
|------|--------|
| 対応ユーザー数 | **1億人以上** |
| 対応組織数 | **100万組織以上** |
| 同時接続数 | 10万接続以上 |
| レスポンス時間 | 管理画面：2秒以内 |

### 1.2 スケーラビリティ設計原則

1. **ページネーション必須**: 全一覧はページ単位（20-100件/ページ）で取得
2. **インデックス最適化**: 検索・ソート対象カラムには必ずインデックス設定
3. **遅延読み込み**: 詳細情報は必要時にのみ取得（N+1問題回避）
4. **キャッシュ活用**: 統計情報は定期更新キャッシュを使用
5. **非同期処理**: 重い処理（CSV出力等）はバックグラウンド実行

### 2.1 権限レベル定義

| レベル | 権限名 | role値 | 説明 | アクセス可能画面 |
|--------|--------|--------|------|------------------|
| **Lv0** | システム開発者 | `developer` | 全システム管理、全権限 | `/system/*`, `/admin/*` |
| **Lv1** | 組織管理者 | `org_admin` | 自組織のメンバー・グループ管理 | `/admin/*`（自組織のみ） |
| **Lv2** | グループ管理者 | `group_admin` | 担当グループのみ管理 | グループ設定のみ |
| **Lv3** | 一般ユーザー | `user` | 通常利用のみ | チャット機能のみ |

## 5. データベース設計

### 5.1 新規テーブル

#### organizations（組織）

```sql
CREATE TABLE organizations (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    name VARCHAR(100) NOT NULL COMMENT '組織名',
    type ENUM('corporate', 'family', 'community', 'other') DEFAULT 'corporate' COMMENT '種別',
    description TEXT COMMENT '説明',
    owner_user_id INT UNSIGNED COMMENT '代表者ID',
    status ENUM('active', 'suspended', 'deleted') DEFAULT 'active' COMMENT '状態',
    max_members INT UNSIGNED DEFAULT 1000 COMMENT '最大メンバー数',
    settings JSON COMMENT '組織設定（JSON）',
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    INDEX idx_org_name (name),
    INDEX idx_org_type (type),
    INDEX idx_org_status (status),
    INDEX idx_org_owner (owner_user_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

#### user_violations（違反履歴）

```sql
CREATE TABLE user_violations (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    user_id INT UNSIGNED NOT NULL COMMENT '対象ユーザーID',
    reported_by INT UNSIGNED COMMENT '通報者ID（NULL=システム検知）',
    organization_id INT UNSIGNED COMMENT '発生組織ID',
    
    violation_type ENUM('spam', 'harassment', 'inappropriate', 'fraud', 'impersonation', 'other') NOT NULL COMMENT '違反種別',
    description TEXT COMMENT '詳細説明',
    evidence_path VARCHAR(500) COMMENT '証拠ファイルパス',
    
    status ENUM('pending', 'investigating', 'resolved', 'dismissed') DEFAULT 'pending' COMMENT '対応状態',
    action_taken ENUM('none', 'warning', 'suspended', 'deleted') DEFAULT 'none' COMMENT '実施した処分',
    action_note TEXT COMMENT '処分メモ',
    handled_by INT UNSIGNED COMMENT '対応した開発者ID',
    
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    resolved_at DATETIME COMMENT '解決日時',
    
    INDEX idx_violation_user (user_id),
    INDEX idx_violation_status (status),
    INDEX idx_violation_type (violation_type),
    INDEX idx_violation_created (created_at DESC),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

#### system_stats（統計キャッシュ）

```sql
CREATE TABLE system_stats (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,

    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    INDEX idx_stat_key (stat_key)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## 6. 大規模対応設計

### 6.1 1億ユーザー対応のポイント

| 課題 | 対策 |
|------|------|
| COUNT(*)の遅延 | system_statsテーブルでキャッシュ（定期更新） |
| 一覧取得の遅延 | 必ずLIMIT/OFFSETを使用、最大100件/ページ |
| 検索の遅延 | FULLTEXTインデックス or Elasticsearch導入（将来） |
| ソートの遅延 | ソート対象カラムにはすべてインデックス |
| JOINの遅延 | 必要最小限のJOIN、詳細は別APIで取得 |

### 6.2 統計更新バッチ（推奨）

```php
// cron: 5分ごとに実行
// /nine/system/cron/update_stats.php

$stats = [
    'total_users' => "SELECT COUNT(*) FROM users WHERE status != 'deleted'",
    'total_organizations' => "SELECT COUNT(*) FROM organizations WHERE status != 'deleted'",
    'active_users_7d' => "SELECT COUNT(*) FROM users WHERE last_login >= DATE_SUB(NOW(), INTERVAL 7 DAY)",
    'suspended_users' => "SELECT COUNT(*) FROM users WHERE status = 'suspended'",
    'pending_violations' => "SELECT COUNT(*) FROM user_violations WHERE status = 'pending'"
];

foreach ($stats as $key => $sql) {
    $value = $pdo->query($sql)->fetchColumn();
    $pdo->prepare("UPDATE system_stats SET stat_value = ? WHERE stat_key = ?")->execute([$value, $key]);
}
```

## 8. セキュリティ要件

| 項目 | 対策 |
|------|------|
| 認証 | システム開発者は特別な認証（role='developer'） |
| アクセス制限 | /system/* はdeveloperのみアクセス可 |
| 操作ログ | 全操作を監査ログに記録 |
| CSRF | 全フォームにCSRFトークン必須 |
| XSS | 出力時にhtmlspecialchars必須 |
| SQLi | プリペアドステートメント必須 |
