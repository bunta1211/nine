# ユニバーサルマッチング機能

## 概要

**「あらゆる需要（欲求）と供給（欲求）を結びつける」** ことで収益を生み出す、Social9の核心機能。

従来のマッチングサービスが「仕事」や「恋愛」など特定分野に限定されるのに対し、
ユニバーサルマッチングは **あらゆる欲求** を対象とします。

---

## 🎯 コンセプト

### 従来のサービス vs ユニバーサルマッチング

| 従来 | ユニバーサルマッチング |
|------|----------------------|
| 仕事マッチング（タイミー等） | 仕事も対象 |
| 恋愛マッチング（Pairs等） | 恋愛も対象 |
| スキルシェア（ココナラ等） | スキルも対象 |
| 売買（メルカリ等） | 売買も対象 |
| **→ 縦割り** | **→ 何でも対象** |

### 基本の流れ

```
┌──────────────┐        ┌──────────────┐
│    需要者    │        │    供給者    │
│  「〇〇したい」│  ←→   │ 「〇〇できます」│
└──────────────┘        └──────────────┘
         ↓                     ↓
    リクエスト投稿          オファー送信
         ↓                     ↓
    ┌──────────────────────────┐
    │       マッチング成立       │
    │    チャットで詳細調整      │
    │        取引実行           │
    │      評価・レビュー        │
    └──────────────────────────┘
         ↓
    手数料3%〜5%（50万人達成後）
```

## 📝 リクエスト（需要）

### リクエストの種類

| 種類 | 説明 | 例 |
|------|------|-----|
| **サービス依頼** | 何かをやってほしい | 引っ越し手伝い、家事代行 |
| **スキル依頼** | 専門知識が必要 | プログラミング、デザイン |
| **モノの購入** | 何かを買いたい | 中古家具、手作り品 |
| **体験・参加** | イベント等に参加したい | 料理教室、ワークショップ |
| **相談・アドバイス** | 話を聞いてほしい | 進路相談、法律相談 |
| **人探し** | 一緒に何かする人 | ゴルフ仲間、勉強仲間 |

### 供給者タイプ

| タイプ | 説明 | 認証要件 |

## 🤝 マッチングの流れ

### 1. リクエスト投稿

```
需要者 → リクエスト投稿
         ↓
    [自動] 条件に合う供給者に通知
```

### 2. オファー受付

```
供給者 → オファー送信（複数可）
         ↓
需要者 ← オファー一覧を確認
```

### 4. 承諾・チャット開始

```
需要者 → オファーを承諾
         ↓
    [自動] 専用チャットルームを作成
         ↓
    詳細をチャットで調整
```

### 5. 取引実行

```
当日、サービス実行
         ↓
支払い（現金 or エスクロー）
         ↓
相互評価
```

## 💰 収益モデル

### 手数料体系

| 取引タイプ | 手数料 | 負担者 | 開始時期 |
|-----------|:------:|:------:|----------|
| 個人→個人 | 3% | 供給者 | 50万人達成後 |
| 個人→法人 | 3% | 供給者 | 50万人達成後 |
| 法人→個人（仕事） | 5% | 法人 | 50万人達成後 |

### 手数料計算例

```
取引金額: ¥10,000
手数料(3%): ¥300
供給者受取: ¥9,700
```

### 50万人達成前

```
手数料: 0%（無料）
目的: ユーザー獲得・取引実績の蓄積
```

## 🔒 安全対策

### 本人確認必須

マッチング機能を使うには **本人確認（Level 3）** が必要。

| 確認項目 | 方法 |
|----------|------|
| 身分証明書 | 運転免許証、マイナンバーカード等 |
| 顔写真照合 | 身分証と顔写真の照合 |
| 住所確認 | 郵送確認（任意） |

### エスクロー決済（将来実装）

```
需要者 → 支払い → [Social9が一時預かり] → 完了後 → 供給者へ
```

- 取引完了まで代金を預かり
- トラブル時は返金対応
- 50万人達成後に実装

## 🗄️ データベース設計

### requests テーブル（リクエスト）

```sql
CREATE TABLE requests (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    user_id INT UNSIGNED NOT NULL,
    
    -- 内容
    title VARCHAR(200) NOT NULL,
    description TEXT NOT NULL,
    category VARCHAR(50) NOT NULL,
    
    -- 条件
    prefecture VARCHAR(20),
    city VARCHAR(50),
    budget INT UNSIGNED COMMENT '予算（円）',
    
    -- 日時
    preferred_date DATE,
    preferred_time_start TIME,
    preferred_time_end TIME,
    
    -- 設定
    urgency ENUM('normal', 'soon', 'urgent') DEFAULT 'normal',
    visibility ENUM('public', 'local', 'organization') DEFAULT 'public',
    
    -- ステータス
    status ENUM('draft', 'open', 'has_offers', 'negotiating', 'matched', 'completed', 'cancelled') DEFAULT 'draft',
    
    -- マッチング
    matched_offer_id INT UNSIGNED,
    matched_at DATETIME,
    completed_at DATETIME,
    
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    INDEX idx_status (status),
    INDEX idx_category (category),
    INDEX idx_area (prefecture, city)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### providers テーブル（供給者プロフィール）

```sql
CREATE TABLE providers (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    user_id INT UNSIGNED NOT NULL UNIQUE,
    
    -- タイプ
    provider_type ENUM('individual', 'business') NOT NULL DEFAULT 'individual',
    
    -- 法人情報
    business_name VARCHAR(100),
    business_address TEXT,
    
    -- プロフィール
    display_name VARCHAR(100) NOT NULL,
    bio TEXT,
    profile_image VARCHAR(500),
    
    -- 対応可能
    categories JSON COMMENT '対応可能カテゴリ',
    service_areas JSON COMMENT '対応エリア',
    
    -- 認証
    is_verified TINYINT(1) DEFAULT 0,
    is_certified_pro TINYINT(1) DEFAULT 0,
    
    -- 統計
    total_completed INT UNSIGNED DEFAULT 0,
    average_rating DECIMAL(2,1) DEFAULT 0.0,
    
    -- ステータス
    status ENUM('pending', 'active', 'suspended') DEFAULT 'pending',
    
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### offers テーブル（オファー）

```sql
CREATE TABLE offers (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    request_id INT UNSIGNED NOT NULL,
    provider_id INT UNSIGNED NOT NULL,
    
    -- 提案内容
    price INT UNSIGNED NOT NULL COMMENT '提案価格',
    message TEXT NOT NULL,
    available_date DATE,
    available_time_start TIME,
    available_time_end TIME,
    
    -- ステータス
    status ENUM('pending', 'viewed', 'accepted', 'rejected', 'withdrawn') DEFAULT 'pending',
    
    viewed_at DATETIME,
    responded_at DATETIME,
    
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    
    FOREIGN KEY (request_id) REFERENCES requests(id) ON DELETE CASCADE,
    FOREIGN KEY (provider_id) REFERENCES providers(id) ON DELETE CASCADE,
    UNIQUE KEY unique_offer (request_id, provider_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### reviews テーブル（評価）

```sql
CREATE TABLE reviews (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    
    -- 対象
    request_id INT UNSIGNED NOT NULL,
    reviewer_id INT UNSIGNED NOT NULL,
    reviewee_id INT UNSIGNED NOT NULL,
    
    -- 評価
    overall_rating TINYINT UNSIGNED NOT NULL COMMENT '1-5',
    punctuality_rating TINYINT UNSIGNED COMMENT '時間厳守',
    quality_rating TINYINT UNSIGNED COMMENT '品質',
    communication_rating TINYINT UNSIGNED COMMENT 'コミュニケーション',
    comment TEXT,
    
    -- 反論
    rebuttal TEXT,
    rebuttal_at DATETIME,
    
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    
    FOREIGN KEY (request_id) REFERENCES requests(id) ON DELETE CASCADE,
    FOREIGN KEY (reviewer_id) REFERENCES users(id) ON DELETE CASCADE,
    FOREIGN KEY (reviewee_id) REFERENCES users(id) ON DELETE CASCADE,
    UNIQUE KEY unique_review (request_id, reviewer_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```
