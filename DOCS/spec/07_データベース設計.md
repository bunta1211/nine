## 👤 ユーザー関連

### users

```sql
CREATE TABLE users (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    
    -- 認証
    email VARCHAR(255) NOT NULL UNIQUE,
    password_hash VARCHAR(255) NOT NULL,
    phone VARCHAR(20),
    
    -- 認証レベル
    auth_level TINYINT UNSIGNED DEFAULT 1 COMMENT '1:メール 2:電話 3:本人確認',
    email_verified_at DATETIME,
    phone_verified_at DATETIME,
    identity_verified_at DATETIME,
    
    -- メール認証用
    email_verification_token VARCHAR(100),
    
    -- パスワードリセット用
    password_reset_token VARCHAR(100),
    password_reset_expires DATETIME,
    
    -- 電話認証用
    phone_verification_code VARCHAR(6),
    phone_verification_expires DATETIME,
    
    -- プロフィール
    display_name VARCHAR(50) NOT NULL,
    avatar_path VARCHAR(500),
    bio TEXT,
    birth_date DATE NOT NULL,
    is_minor TINYINT(1) DEFAULT 0,
    
    -- 地域
    prefecture VARCHAR(20),
    city VARCHAR(50),
    
    -- オンライン状態
    online_status ENUM('online', 'away', 'busy', 'offline') DEFAULT 'offline',
    last_seen DATETIME,
    
    -- 言語設定
    display_language VARCHAR(10) DEFAULT 'ja',
    translate_to VARCHAR(10) DEFAULT 'ja',
    auto_translate TINYINT(1) DEFAULT 1,
    
    -- 権限
    role ENUM('user', 'org_admin', 'system_admin') DEFAULT 'user',
    
    -- 状態
    status ENUM('active', 'suspended', 'deleted') DEFAULT 'active',
    
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    -- インデックス
    INDEX idx_email (email),
    INDEX idx_status (status),
    INDEX idx_display_name (display_name),
    INDEX idx_prefecture_city (prefecture, city),
    INDEX idx_online (online_status, last_seen)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## 🏢 ソーシャルグループ関連

> **ソーシャルグループ (Social Groups)**: 家族、老人会、子ども会、青年会議所、株式会社等のまとまった集団
> テーブル名は互換性のため `organizations` を維持
> メンバーには「窓口」「代表」「事務局」等の肩書(title)を設定可能

### organizations（ソーシャルグループ）

```sql
CREATE TABLE organizations (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    
    name VARCHAR(100) NOT NULL,
    type ENUM('corporation', 'family', 'school', 'group') NOT NULL,
    description TEXT,
    icon_path VARCHAR(500),
    
    default_member_role ENUM('member', 'restricted') DEFAULT 'member',
    require_admin_approval TINYINT(1) DEFAULT 0,
    
    created_by INT UNSIGNED NOT NULL,
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    updated_at DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    
    FOREIGN KEY (created_by) REFERENCES users(id),
    
    -- インデックス
    INDEX idx_type (type),
    INDEX idx_name (name)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

### organization_members（ソーシャルグループメンバー）

```sql
CREATE TABLE organization_members (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    organization_id INT UNSIGNED NOT NULL,
    user_id INT UNSIGNED NOT NULL,
    
    role ENUM('owner', 'admin', 'member', 'restricted') DEFAULT 'member',
    
    -- 肩書（窓口、代表、事務局 等）
    title VARCHAR(50) DEFAULT NULL,
    
    -- 制限設定（restricted用）
    external_contact ENUM('none', 'org_only', 'approved', 'notify', 'allow') DEFAULT 'allow',
    call_restriction ENUM('none', 'group_only', 'approved', 'allow') DEFAULT 'allow',
    file_send_allowed TINYINT(1) DEFAULT 1,
    
    -- 利用時間制限
    usage_start_time TIME,
    usage_end_time TIME,
    daily_limit_minutes INT UNSIGNED,
    
    -- 監視設定
    can_view_messages TINYINT(1) DEFAULT 0,
    can_view_online TINYINT(1) DEFAULT 1,
    can_view_contacts TINYINT(1) DEFAULT 1,
    
    joined_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    left_at DATETIME,
    
    FOREIGN KEY (organization_id) REFERENCES organizations(id) ON DELETE CASCADE,
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE,
    
    -- インデックス
    UNIQUE KEY unique_member (organization_id, user_id),
    INDEX idx_user (user_id),
    INDEX idx_role (organization_id, role)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

## 🔍 効率的なクエリ例

### 会話一覧取得（N+1問題回避）

```sql
SELECT 
    c.id,
    c.type,
    c.name,
    c.icon_path,
    m.content AS last_message,
    m.created_at AS last_message_at,
    m.sender_id AS last_sender_id,
    u.display_name AS last_sender_name,
    (
        SELECT COUNT(*) 
        FROM messages 
        WHERE conversation_id = c.id 
        AND created_at > COALESCE(cm.last_read_at, '1970-01-01')
        AND deleted_at IS NULL
    ) AS unread_count
FROM conversations c
INNER JOIN conversation_members cm 
    ON cm.conversation_id = c.id 
    AND cm.user_id = :user_id 
    AND cm.left_at IS NULL
LEFT JOIN messages m ON m.id = (
    SELECT id FROM messages 
    WHERE conversation_id = c.id AND deleted_at IS NULL
    ORDER BY created_at DESC 
    LIMIT 1
)
LEFT JOIN users u ON u.id = m.sender_id
ORDER BY COALESCE(m.created_at, c.created_at) DESC;
```

### メンション検索

```sql
-- 自分がメンションされたメッセージ一覧
SELECT 
    m.*,
    c.name AS conversation_name,
    u.display_name AS sender_name
FROM message_mentions mm
INNER JOIN messages m ON m.id = mm.message_id
INNER JOIN conversations c ON c.id = m.conversation_id
INNER JOIN users u ON u.id = m.sender_id
WHERE mm.mentioned_user_id = :user_id
ORDER BY mm.created_at DESC
LIMIT 50;
```

### リクエスト検索（マッチング）

```sql
-- エリア＋カテゴリで検索
SELECT 
    r.*,
    u.display_name AS requester_name,
    u.avatar_path AS requester_avatar,
    (SELECT COUNT(*) FROM offers WHERE request_id = r.id) AS offer_count
FROM requests r
INNER JOIN users u ON u.id = r.user_id
WHERE r.status = 'open'
AND r.prefecture = :prefecture
AND (:city IS NULL OR r.city = :city)
AND (:category IS NULL OR r.category = :category)
ORDER BY 
    CASE r.urgency 
        WHEN 'urgent' THEN 1 
        WHEN 'soon' THEN 2 
        ELSE 3 
    END,
    r.created_at DESC
LIMIT 20;
```

### プロバイダー検索（usersをJOIN）

```sql
SELECT

FROM providers p
INNER JOIN users u ON u.id = p.user_id
```

### user_violations（違反履歴）

問題ユーザーの通報・違反履歴を管理。

```sql
CREATE TABLE user_violations (
    id INT UNSIGNED AUTO_INCREMENT PRIMARY KEY,
    user_id INT UNSIGNED NOT NULL COMMENT '対象ユーザーID',
    reported_by INT UNSIGNED COMMENT '通報者ID（NULL=システム検知）',
    organization_id INT UNSIGNED COMMENT '発生組織ID',
    
    violation_type ENUM('spam', 'harassment', 'inappropriate', 'fraud', 'impersonation', 'other') NOT NULL COMMENT '違反種別',
    description TEXT COMMENT '詳細説明',
    evidence_path VARCHAR(500) COMMENT '証拠ファイルパス',
    
    status ENUM('pending', 'investigating', 'resolved', 'dismissed') DEFAULT 'pending' COMMENT '対応状態',
    action_taken ENUM('none', 'warning', 'suspended', 'deleted') DEFAULT 'none' COMMENT '実施した処分',
    action_note TEXT COMMENT '処分メモ',
    handled_by INT UNSIGNED COMMENT '対応した開発者ID',
    
    created_at DATETIME DEFAULT CURRENT_TIMESTAMP,
    resolved_at DATETIME COMMENT '解決日時',
    
    INDEX idx_violation_user (user_id),
    INDEX idx_violation_status (status),
    INDEX idx_violation_type (violation_type),
    INDEX idx_violation_created (created_at DESC),
    FOREIGN KEY (user_id) REFERENCES users(id) ON DELETE CASCADE
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COLLATE=utf8mb4_unicode_ci;
```

| カラム | 型 | 説明 |
|--------|-----|------|
| id | INT UNSIGNED | 主キー |
| user_id | INT UNSIGNED | 対象ユーザー |
| reported_by | INT UNSIGNED | 通報者（NULL=システム） |
| violation_type | ENUM | spam/harassment/inappropriate/fraud/impersonation/other |
| status | ENUM | pending/investigating/resolved/dismissed |
| action_taken | ENUM | none/warning/suspended/deleted |

## 📈 大規模対応設計（1億ユーザー）

### パフォーマンス原則

| 課題 | 対策 |
|------|------|
| COUNT(*)の遅延 | system_statsテーブルでキャッシュ |
| 一覧取得の遅延 | 必ずLIMIT/OFFSETを使用（最大100件） |
| 検索の遅延 | 検索カラムにインデックス必須 |
| ソートの遅延 | ソート対象にインデックス必須 |
| JOINの遅延 | 必要最小限、詳細は別API |

### 推奨クエリパターン

```sql
-- ❌ 悪い例：COUNT(*)直接実行
SELECT COUNT(*) FROM users; -- 1億件で数秒かかる

-- ✅ 良い例：キャッシュテーブル使用
SELECT stat_value FROM system_stats WHERE stat_key = 'total_users';

-- ❌ 悪い例：OFFSET大きすぎ
SELECT * FROM users LIMIT 20 OFFSET 1000000; -- 遅い

-- ✅ 良い例：カーソルベース
SELECT * FROM users WHERE id > :last_id ORDER BY id LIMIT 20;
```
