# 複数アカウント同時ログインガイド

KENとシステム管理者など、複数アカウントを別タブ/別ウィンドウで同時にログインして会話テストする方法です。

## 問題

同じブラウザの複数タブは**セッション（Cookie）を共有**しています。そのため：

1. タブ1で KEN でログイン
2. タブ2で システム管理者 でログイン → セッションが上書きされる
3. タブ1を更新すると システム管理者 のアカウントになる

## 解決方法

### 方法1: 別のブラウザを使う（推奨・最簡単）

- **Chrome** で KEN にログイン
- **Firefox** または **Edge** で システム管理者 にログイン

それぞれ別のブラウザは Cookie を共有しないため、両方ともログイン状態を維持できます。

### 方法2: シークレット/プライベートモードを使う

- **通常ウィンドウ** で KEN にログイン
- **シークレットウィンドウ**（Chrome: Ctrl+Shift+N）で システム管理者 にログイン

シークレットモードは通常モードと Cookie を共有しないため、両方ともログイン状態を維持できます。

### 方法3: サブドメインを使う（本番環境）

social9.jp で以下のサブドメインを設定すると、それぞれ**別セッション**でログインできます。

1. **DNS設定**: ken.social9.jp と sys.social9.jp をサーバーに振り向ける
2. **Webサーバー**: 両ホスト名で同じアプリを提供
3. **ログイン**:
   - https://ken.social9.jp/ で KEN にログイン
   - https://sys.social9.jp/ で システム管理者 にログイン

セッション Cookie はホスト毎に分離されるため、両方ともログイン状態を維持できます。
