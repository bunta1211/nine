# 携帯電話機能のデプロイ手順

サーバーへ「携帯電話の登録・検索」機能を反映するときの、**ファイルの送り方**と**SQLの実行の仕方**をまとめています。

---

## 手順の流れ（概要）

1. **SQLを先に実行する**（データベースに `phone` カラムを追加）
2. **変更したファイルをサーバーに送る**
3. **動作確認する**

※ 本番サーバーでは、可能ならバックアップを取ってから行ってください。

---

## ステップ1：SQLをデータベースに登録する

### 1-1. 実行するSQLの内容

次の**2文を1つずつ**、**本番で使っているデータベース**に対して実行します。  
（MySQL 5.7 や MariaDB 10.x でも実行できる構文です。）  
コピーするときは、**1つ目のブロックだけ**をコピーして実行 → 成功したら**2つ目のブロックだけ**をコピーして実行してください。

**1つ目（カラム追加）** — これをコピーして実行

```sql
ALTER TABLE users ADD COLUMN phone VARCHAR(20) DEFAULT NULL COMMENT '携帯電話番号（数字のみ）' AFTER password_hash;
```

**2つ目（インデックス追加）** — 1つ目が成功したら、これをコピーして実行

```sql
ALTER TABLE users ADD INDEX idx_phone (phone);
```

- **1回目**：両方とも「Query OK」などと出れば成功です。
- **2回目以降**（既に実行済みのとき）：  
  「Duplicate column name 'phone'」や「Duplicate key name 'idx_phone'」と出ることがあります。その場合は**既に適用済み**なので、そのまま完了として問題ありません。

### 1-2. 実行方法（よくある2通り）

#### 方法A：phpMyAdminで実行する

1. サーバーのphpMyAdmin（または同じDBを触れる管理画面）にログインする。
2. 左で **Social9用のデータベース** を選択する。
3. 上のタブで **「SQL」** をクリックする。
4. 上のSQLをそのまま貼り付ける。
5. **「実行」**（または「Go」）をクリックする。
6. 「〇件のクエリを実行しました」などと出ればOK。  
   「カラムは既に存在します」などのメッセージが出ても、この機能のためには問題ありません。

#### 方法B：MySQLのコマンドラインで実行する

1. サーバーにSSHで入れる場合、MySQLクライアントでDBに接続する。
2. 次のように実行する（データベース名は環境に合わせて変える）。

```bash
mysql -u ユーザー名 -p データベース名
```

パスワードを聞かれたら入力する。

3. プロンプトが出たら、次の**1つ目**をコピーして実行する。

```sql
ALTER TABLE users ADD COLUMN phone VARCHAR(20) DEFAULT NULL COMMENT '携帯電話番号（数字のみ）' AFTER password_hash;
```

4. 成功したら、**2つ目**をコピーして実行する。

```sql
ALTER TABLE users ADD INDEX idx_phone (phone);
```

5. 両方とも `Query OK` などと表示されれば成功。

### 1-3. 実行結果の確認（任意）

- phpMyAdminなら：左の一覧で **「users」** テーブルを開き、構造に **「phone」** カラムがあるか確認する。
- コマンドラインなら：  
  `DESCRIBE users;` で `phone` が一覧に出ていればOK。

ここまでで「SQLの登録」は完了です。

## ステップ2：サーバーに送るファイル一覧

次のファイルを、**ローカルとサーバーで同じパス**になるようにアップロード（またはデプロイ）します。

### 変更したファイル（上書きで送る）

| サーバー上のパス | 説明 |
|-----------------|------|
| `register.php` | 新規登録フォームに携帯電話欄を追加 |
| `includes/auth/Auth.php` | 登録時に電話番号を保存する処理を追加 |
| `api/users.php` | ユーザー検索で携帯電話でも検索できるように変更 |
| `settings.php` | 設定画面に携帯電話の表示・編集を追加 |
| `includes/lang.php` | 「携帯電話」などの文言を追加 |
| `includes/chat/topbar.php` | （以前の修正）メモのツールチップを「メモ」に変更 |
| `database/DEPENDENCIES.md` | phone の説明を追記 |
| `api/DEPENDENCIES.md` | users.php の検索説明を追記 |

### 新規で追加したファイル（そのまま送る）

| サーバー上のパス | 説明 |
|-----------------|------|
| `database/migration_user_phone.sql` | 上記SQLをまとめたマイグレーション用ファイル（参考・再実行用） |
| `DOCS/携帯電話機能のデプロイ手順.md` | この手順書 |

※ `migration_user_phone.sql` は「手順の参考用・同じSQLを再度実行したいとき用」です。ステップ1で既にSQLを実行済みなら、**必須ではありません**が、置いておくと運用時に便利です。

## ステップ3：ファイルの送り方（やり方の例）

### 方法1：FTP / SFTP クライアント（FileZilla など）で送る

1. **FileZilla** や **WinSCP** などで、サーバーに接続する。
2. **左側（ローカル）** で、プロジェクトのフォルダ（例：`c:\xampp\htdocs\nine`）を開く。
3. **右側（サーバー）** で、本番のドキュメントルート（例：`/home/xxx/public_html` や `www`）を開く。
4. 左側で次のファイル／フォルダを選び、右側の**同じ相対パス**にドラッグ＆ドロップ（またはアップロード）する。
   - `register.php` → サーバーの `register.php`
   - `includes/auth/Auth.php` → サーバーの `includes/auth/Auth.php`
   - 同様に、一覧の他のファイルも **パスをそろえて** 送る。
5. 上書き確認が出たら「上書き」でOK。  
   新規ファイル（`migration_user_phone.sql`、この手順書）は、右側で同じフォルダ構成を作ってから送る。

### 方法2：Git でデプロイしている場合

1. ローカルでコミット・プッシュまで済ませる。
2. サーバー側で `git pull` する（または、Git連携の自動デプロイなら、プッシュ後に自動反映される想定）。
3. サーバー上のパスが、上記「送るファイル一覧」と同じになっているか確認する。

### 方法3：ZIPでまとめて送ってから解凍する

1. ローカルで、変更・追加したファイルだけを**同じフォルダ構成で**ZIPにまとめる。
2. サーバーにZIPをアップロードする（FTP/SFTPや管理画面のファイルマネージャなど）。
3. サーバー上でZIPを解凍し、中身を本番のドキュメントルートに**上書きコピー**する。  
   （既存ファイルを消さないよう、解凍先のパスに注意する。）

## ステップ4：動作確認

1. **新規登録**
   - 未登録のメールで新規登録画面を開く。
   - 「携帯電話（任意）」欄が表示され、入力・保存できること。

2. **設定画面**
   - ログイン後、設定 → 基本設定を開く。
   - 「携帯電話」欄が表示され、保存できること。

3. **検索**
   - チャットなどで「ユーザーを検索」する機能を開く。
   - 携帯電話番号の一部（例：09012345678 の下4桁など）で検索し、該当ユーザーが出ること。

4. **既存ユーザー**
   - 既存アカウントでログインし、設定で携帯電話を追加・変更できること。
   - その番号で検索できること。

---

## トラブル時の確認ポイント

- **「Unknown column 'phone'」などと出る**  
  → ステップ1のSQLが実行されていないか、別のDBを見ている可能性があります。  
  本番で使っているデータベースに対して、もう一度SQLを実行してください。

- **登録・設定で「この携帯電話番号は既に登録されています」**  
  → 同じ番号が別ユーザーで既に登録されています。別の番号を入力するか、重複を解消してからにしてください。

- **検索でヒットしない**  
  → 検索対象ユーザーが「携帯電話」を設定しているか、設定画面で確認してください。  
  番号は数字のみで保存されるため、「090-1234-5678」でも「09012345678」でも検索可能です。

## まとめ

| 順番 | やること |
|------|----------|
| 1 | 本番DBで、`users` に `phone` カラムと `idx_phone` を追加するSQLを実行する |
| 2 | 変更・追加したファイルを、サーバーの同じパスに送る（FTP/Git/ZIPなど） |
| 3 | 新規登録・設定・検索で携帯電話が使えるか確認する |

SQLを先に実行し、そのあとでファイルを送る順番を守ると、エラーが出にくくなります。
