# 長文ペースト時のサーバーエラー（500）対策

## 概要
Chatwork 等から大量ログをコピー＆ペーストして送信すると、`POST /api/messages.php` が 500 Internal Server Error（`error_type: fatal`）になる問題への対応。

## 原因
- **1000文字以上**で長文を PDF に変換する処理で、**極端に長い文字列**を TCPDF に渡すとメモリ不足やタイムアウトで PHP が致命的エラーになる
- PDF 変換に失敗した場合でも、**本文をそのまま DB の `content`（TEXT＝約64KB）に保存しようとすると**データが長すぎてエラーになる

## 実施した修正（api/messages.php）

### 1. 送信メッセージの文字数上限
- **1,000万文字**を上限に設定（`$maxContentChars = 10000000`）
- 超過時は `errorResponse('メッセージは1,000万文字までです。長いログは分割して送信してください。', 400)` で返す

### 2. PDF 変換の対象上限
- **15万文字**を超える長文は PDF 変換を試みず、事前にエラーで弾く（`$maxPdfChars = 150000`）
- 「長文は自動でPDFに変換されますが、15万文字を超えると変換できません。150,000文字以内に分割して送信してください。」と返す
- これにより、メモリ不足によるサーバーエラーを防ぐ

### 3. PDF 変換の try-catch
- `textToPdf()` を **try-catch（Throwable）** で囲む
- 例外発生時（メモリ不足等）はログを書き、**1万文字超**の場合は「長文の処理中にエラーが発生しました。15万文字以内に分割して送信してください。」と 400 で返す
- 1万文字未満の場合は従来どおり本文のまま送信を試行

### 4. DB の content 長制限
- `content` は **TEXT（約64KB）** のため、PDF 変換せずに本文をそのまま保存する場合は **約65,000バイト** を上限にチェック
- 超過時は「文字数が多すぎます（長文はPDFに自動変換されますが、変換に失敗した場合は約6万5千文字以内に分割して送信してください）。」と返す

## サーバーに送るファイル

| ファイル | 内容 |
|----------|------|
| **api/messages.php** | 上記の長文上限・PDF変換上限・try-catch・content 長チェック |

## 運用メモ
- 15万文字を超えるログは**複数メッセージに分割**して送信するようユーザーに案内する
- サーバーの `memory_limit` や `max_execution_time` を必要に応じて見直す（PDF 変換は 15 万文字以内に抑えているが、環境によってはさらに下げる余地あり）
