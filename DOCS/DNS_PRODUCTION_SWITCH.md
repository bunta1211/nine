# 本番環境（social9.jp）に反映されない場合の対処

---

## 原因

**social9.jp がまだ旧サーバー（heteml）を指している**ため、EC2 への変更が本番に反映されません。

| アクセス先 | 現在のIP | 状態 |
|------------|----------|------|
| social9.jp | **157.7.189.251**（旧 heteml） | 変更が反映されない |
| 54.95.86.79 | **54.95.86.79**（EC2） | 変更が反映済み |

---

## 解決手順

### 1. Route 53 の A レコードを確認

1. **AWS コンソール** → **Route 53** → **ホストゾーン** → **social9.jp**
2. **レコード** 一覧で **A レコード**（レコード名が空欄または `@`）を確認
3. **値** が `54.95.86.79` になっているか確認
4. 違う場合は **編集** し、`54.95.86.79` に変更して保存

### 2. ムームードメインのネームサーバーを確認

1. **https://muumuu-domain.com/** にログイン
2. **ドメイン管理** → **social9.jp** → **ネームサーバー設定変更**
3. **Route 53 のネームサーバー** が設定されているか確認：
   - `ns-76.awsdns-09.com.`
   - `ns-712.awsdns-25.net.`
   - `ns-1892.awsdns-44.co.uk.`
   - `ns-1149.awsdns-15.org.`
4. 違う場合は上記に変更し、保存

### 3. DNS 反映の確認

PowerShell で実行：

```powershell
nslookup social9.jp
```

**期待する結果**: `54.95.86.79` が表示される

- `157.7.189.251` のまま → ネームサーバー変更が反映されていない（最大48時間待つ）
- `54.95.86.79` → DNS 切り替え完了

---

## 4. あなたの PC で DNS を確認（重要）

**EC2 上**で `nslookup` しても、**あなたのブラウザが使う DNS** とは別です。

**あなたが使っている Windows PC** で PowerShell を開き、次を実行：

```powershell
nslookup social9.jp
```

- **54.95.86.79** が表示される → あなたの PC は EC2 を参照している
- **157.7.189.251** が表示される → あなたの PC はまだ旧サーバーを参照している

**157.7.189.251 の場合** → social9.jp は旧サーバーに接続しているため、変更が反映されません。

**対処**：PC で `ipconfig /flushdns` を実行（管理者権限）し、ブラウザを再起動してから再度試す。

---

## 5. server-check.php でサーバーを確認

1. **topbar.php** と **server-check.php** を EC2 にアップロード
2. ブラウザで以下を開く（シークレットモード推奨）：
   - `http://54.95.86.79/api/server-check.php`
   - `http://social9.jp/api/server-check.php`
3. レスポンスを比較：
   - **同じ内容**（`topbar_has_test_badge: true` など） → 同じサーバー。ブラウザキャッシュの可能性
   - **social9.jp が 404 や別内容** → social9.jp は旧サーバーを指している

---

## 6. hosts ファイルで強制切り替え（動作確認用）

**nslookup は 54.95.86.79 なのに social9.jp で 404 になる場合**、Chrome のセキュア DNS 等が別の IP を参照している可能性があります。

**一時的に hosts で social9.jp を EC2 に固定**して、動作を確認できます。

### 手順（管理者権限のメモ帳で）

1. **メモ帳** を **管理者として実行**
2. **ファイル** → **開く** → `C:\Windows\System32\drivers\etc\hosts`
3. 末尾に以下を追加して保存：

**⚠️ 重要：ファイル名は `hosts` のまま（`hosts.txt` にしない）**
- 上書き保存する場合は **Ctrl+S** でそのまま保存
- 「名前を付けて保存」する場合は：**ファイルの種類** → **すべてのファイル (\*.\*)**、**ファイル名** → `hosts` のみ

```
54.95.86.79 social9.jp
54.95.86.79 www.social9.jp
```

4. ブラウザを再起動
5. `http://social9.jp/api/server-check.php` を開く

**JSON が表示されれば** → DNS の参照先が原因だったことが確認できます。Chrome のセキュア DNS をオフにするか、しばらく hosts のまま運用できます。

**確認後**、hosts の追加行を削除すれば元に戻ります。

## 7. hosts 変更後に「オフライン」「アクセスできない」になった場合

### 確認1: IP 直接アクセス

まず `http://54.95.86.79/chat.php` を開く（`http://` を明示）。

- **表示される** → EC2 は稼働中。social9.jp の向き先かプロトコルが原因
- **表示されない** → EC2 やネットワークに問題の可能性

### 確認2: HTTPS を使っていないか

EC2 にはまだ SSL が未設定のため、**https://social9.jp** では接続できません。

- ブックマークやアドレスバーが **https** になっていないか確認
- **http://social9.jp/chat.php** を使う（`http://` を明示）

### 確認3: hosts の保存先

編集しているファイルが **hosts.txt** のままの場合、DNS には反映されません。

1. `C:\Windows\System32\drivers\etc\` を開く
2. **hosts**（拡張子なし）を開いて編集
3. 保存は **Ctrl+S** で上書き（「名前を付けて保存」で hosts.txt にしない）

### 一時的に hosts を元に戻す

social9.jp を旧サーバーに戻したい場合、hosts に追加した次の 2 行を削除：

```
54.95.86.79 social9.jp
54.95.86.79 www.social9.jp
```

削除後、social9.jp は再び DNS（旧サーバー）を参照します。

---

## 一時的な確認方法

DNS が切り替わるまで、EC2 の IP で直接アクセスして動作確認できます。

- ログイン: `http://54.95.86.79/index.php`
- チャット: `http://54.95.86.79/chat.php`

## 参考：Route 53 のネームサーバー確認

Route 53 のホストゾーンで **NS レコード** を開くと、使用するネームサーバーが表示されます。ムームードメイン側には、この一覧の値をそのまま設定してください。
