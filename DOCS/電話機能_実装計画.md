# 電話機能 実装計画

## 1. すでに実装している部分

### 1.1 アプリ内通話（VoIP・ビデオ通話）

| 項目 | 内容 |
|------|------|
| **方式** | Jitsi Meet（WebRTC） |
| **種類** | 音声通話・ビデオ通話（DM / グループ） |
| **仕様** | 最大50人、ルーム名で管理、meet.jit.si 使用 |

**主なファイル**

| ファイル | 役割 |
|----------|------|
| `api/calls.php` | 通話作成(create)・参加(join)・退出(leave)・拒否(decline)・アクティブ取得(get_active)・履歴(history) |
| `call.php` | 通話専用画面（Jitsi 埋め込み） |
| `includes/chat/scripts.php` | チャット内の通話メニュー・Jitsi API 初期化・ミュート/カメラ/画面共有/終了 |
| `includes/chat/call-ui.php` | 通話用UI（ビデオバブル・コントロールバー等） |
| `assets/js/chat/call.js` / `chat-call.js` | 通話関連フロント |
| `settings.php` (section=call) | 着信音・カメラ/マイク初期ON・画質・ノイズキャンセル等 |

**DB**

- `calls` … 通話ID・会話ID・開始者・Jitsi room_id・種別(audio/video)・状態・開始/終了時刻・通話時間
- `call_participants` … 通話ごとの参加者・状態(invited/joined/left/declined)
- `user_call_settings` … 着信音・カメラ/マイクデフォルト・画質等
- `organization_members.call_restriction` … 未成年の通話制限（group_only / approved_only 等）

**認証・制限**

- **通話は電話認証を前提にしない**（アプリ内通話のみのため。`api/calls.php` の電話認証チェックは削除済み）。
- 未成年は組織の `call_restriction` に応じて、グループのみ・許可相手のみ等の制限あり。
- 着信通知は `notifications`（type: call_incoming）で保存。

### 1.2 電話番号まわり（認証・プロフィール用）

| 項目 | 内容 |
|------|------|
| **用途** | 本人確認・認証レベル向上（通話機能利用の条件）、検索・プロフィール表示 |
| **DB** | `users.phone` / `phone_verified_at`、`phone_verifications` 等 |

**主なファイル**

- `includes/auth/Auth.php` … `sendPhoneVerification()` / `verifyPhone()`（SMS送信は**未実装・コメントのみ**）
- `settings.php` … 携帯電話番号入力・重複チェック
- `api/messages.php` 等 … メールまたは電話番号でのユーザー検索

**現状の不足**

- 認証コード送信が「SMS送信」のコメントのみで、実際のSMS送信（Twilio 等）は未実装。

## 2. 電話機能の拡張パターン（選択肢）

### 2.1 アプリ内通話の強化（現行の延長）

- **内容**: Jitsi のまま、UI・安定性・多人数・モバイル対応を改善。
- **例**: 着信UI改善、通話履歴の一覧画面、録画・録音の可否ポリシー、自前 Jitsi サーバー検討。
- **コスト**: 小〜中。既存の `api/calls.php` / `call.php` / チャット内通話を拡張する形。

---

### 2.2 PSTN（固定電話・携帯）への発信／着信

- **内容**: アプリから「通常の電話番号」へ発信、または電話からアプリ内の相手へ着信させる。
- **主な方法**:
  - **Twilio** … 音声/SMS API。発信・着信・SMS を一括で扱いやすい。
  - **Vonage (Nexmo)** … 音声/SMS。国際番号が豊富。
  - **AWS Chime SDK / Amazon Connect** … 音声・会議。AWS 中心なら検討可。
  - **WebRTC to PSTN ゲートウェイ** … Jitsi に PSTN ゲートウェイを繋ぎ、電話番号でルームに参加させる方式。
- **実装イメージ**:
  - チャットやプロフィールの「電話番号」横に「発信」ボタンを追加。
  - クリック → `api/calls.php` に「PSTN発信」用 action を追加 → Twilio 等で相手番号へ発信。
  - または「電話で参加」用の番号＋PIN を Jitsi ルームに紐づけ、`call.php` で表示。
- **コスト**: 中〜大。月額・通話料・番号取得料がかかる。SMS認証もここで実装すると一石二鳥。

---

### 2.3 SMS認証の実装（電話番号認証の完成）【実装済み】

- **実装内容**: 携帯番号での新規登録・ログインを追加。`config/sms.php`・`includes/SmsSender.php`（log/twilio/sns）、`api/auth_otp.php` の send_code/verify_code/set_password で電話対応。`sms_verification_codes` テーブルでコード管理。詳細は `DOCS/PHONE_REGISTRATION_PROGRESS.md`。
- **方法**:
  - Twilio / AWS SNS を `SmsSender` で利用可能。`config/sms.local.php` で `SMS_DRIVER` を指定。
  - 開発時は `SMS_DRIVER=log` で送信せずログのみ。
- **既存との関係**: `users.email` を NULL 許容に変更し、電話のみ登録ユーザーをサポート。ログインは `api/auth.php` でメール or 電話でユーザー検索。
- **コスト**: 小〜中。SMS 1通あたりの単価と、送信回数制限・レート制限の設計が必要。

---

### 2.4 クリックトゥコール（端末の電話アプリへ渡すだけ）

- **内容**: 画面に表示した電話番号をタップすると、端末の「電話」アプリの発信画面を開く（`tel:` リンク）。
- **実装**: フロントのみ。`<a href="tel:+81...">...</a>` または JS で `location.href = 'tel:' + number`。
- **メリット**: 実装が簡単。PSTN 通話料はユーザーの携帯契約に任せられる。
- **デメリット**: アプリ内の「通話履歴」や「発信元」の管理はできない（端末側で完結）。

---

### 2.5 通話履歴・着信ログの強化

- **内容**: すでに `api/calls.php` の `history` と `calls` / `call_participants` があるので、これを活かす。
- **例**:
  - 設定または専用ページで「通話履歴」一覧（発信/着信/不在の区別、時間・相手・種別）。
  - 未読の「着信通知」を一覧表示・既読化。
- **コスト**: 小。既存 API と DB の拡張で可能。

## 3. 提案（優先度の目安）

| 優先度 | 内容 | 方法 | 備考 |
|--------|------|------|------|
| 高 | 電話番号認証を完成させる | 2.3 SMS認証（Twilio 等） | 既存の「電話認証必須」を通話で使うなら、SMS送信の実装が前提。 |
| 高 | アプリ内通話の安定化・使いやすさ | 2.1 の延長 | 着信UI・履歴表示・エラー時のメッセージなど。 |
| 中 | 固定・携帯への発信が必要な場合 | 2.2 PSTN（Twilio 等） | 料金・番号取得・利用規約の検討が必要。 |
| 低 | 番号表示だけ「かける」体験にしたい場合 | 2.4 クリックトゥコール | フロントのみで実装可能。 |

---

## 4. 次のステップ（例）

1. **SMS認証を入れたい場合**  
   - Twilio（または他SMSプロバイダ）のアカウント・APIキーを用意。  
   - `config/` に SMS 用の設定ファイル（または env）を追加。  
   - `Auth::sendPhoneVerification()` 内で SMS API を呼ぶ処理を実装。  
   - レート制限・テスト用番号・本番番号の切り替えを検討。

2. **PSTN発信を入れたい場合**  
   - 要件整理（発信のみか、着信もか、SMSもか）。  
   - Twilio 等のトライアルで「発信→相手の電話が鳴る」まで検証。  
   - `api/calls.php` に `action=pstn_create` のようなエンドポイントを追加し、トークン・番号・権限チェックを実装。

3. **クリックトゥコールだけ入れる場合**  
   - メンバー一覧・プロフィール・チャット等で「電話番号」を表示している箇所に `tel:` リンクを追加。  
   - プライバシー（番号を誰に見せるか）の設定があれば、それに従って表示/非表示を制御。
