# 検索ポリシー（個人アドレス帳・グループ追加・表示名）

AI秘書・ヘルプで参照するための検索機能のポリシー説明です。

## 総則

- **個人が人を探すとき**（トップバーのグローバル検索）: **名前だけでは検索できません**。メールアドレスまたは携帯番号を入力した場合のみ、該当ユーザーが結果に表示されます。メッセージ・グループの検索は従来どおりキーワードで検索できます。
- **本名・表示名で検索できるのは組織内だけに限定**しています。検索されたくない場合は組織を変える運用を徹底し、AI秘書をはじめユーザーに説明できるようにします。
- **個人アドレス帳**: メールアドレスまたは携帯番号で検索し、登録ユーザーがいる場合は**アドレス追加申請**を送れます。**一致したデータがない場合でも、有効なメールアドレスには招待（招待メール送信）を送る**ことができます。受信者には「〇〇の個人アドレス帳に追加受諾」リンクが届きます。
- **グループ追加**: 自分が所属する**組織（organization_members）**にいる人物のみを検索・選択できます。組織未所属の場合は、従来どおり同じグループのメンバーから選択できます。
- 上記は**携帯版・パソコン版の両方**で同じ条件です。

## 個人アドレス帳の検索

- **検索条件**: メールアドレス（Email）または携帯番号のみ。表示名・本名では検索しません。
- **登録ユーザーに一致した場合**: **アドレス追加申請**を送信できます。
- **一致しなかった場合**: 入力されたメールアドレスが有効な形式であれば、「**招待メール送信**」により招待メールを送信できます。未登録の方にも「〇〇の個人アドレス帳に追加受諾」のリンクが届きます。
- **説明**: 「メールアドレスまたは携帯番号で検索。登録済みの方はアドレス追加申請、未登録のメールアドレスには招待を送れます」をUIに記載しています。

## グループ追加の検索

- **検索対象**: 自分が所属する**組織（organization_members）**にいるユーザーのみ。同じグループにいる人ではなく、同じ組織にいる人をメンバーとして選べます。
- 組織に所属していない場合は、従来どおり「同じグループのメンバー」から選択できます。

## 本名・表示名検索の組織内限定

- 表示名・本名（`display_name` / `full_name`）で検索する機能（例: メッセージ先検索、ユーザー検索）では、**組織内のユーザーにのみ**表示名でのヒットを適用します。組織外のユーザーは表示名ではヒットしません。
- メールアドレス・携帯番号での検索は、従来どおり「同じグループのメンバー」または「システム管理者」を対象にしています。
- これにより「検索されたくない場合は組織を変える」を実現し、AI秘書やヘルプでその旨を説明できます。

## 関連ファイル

- `api/messages.php` … グローバル検索（action=search）。**個人**のユーザー検索ではメール（@含む）または携帯番号（10桁以上）のときのみユーザー結果を返す。システム管理者は従来どおり表示名でも検索可能。
- `api/friends.php` … 個人アドレス帳検索（action=search）、招待送信（send_invite。メール件名・本文は「〇〇の個人アドレス帳に追加受諾」）
- `api/users.php` … グループ追加用検索（for_group_add）、list_group_members、組織内検索（scope=org）。**組織ページ・システム管理ページの検索は従来どおり**（名前・メール・携帯で検索可能）。
- チャット左パネル・個人アドレス帳モーダル（Mailタブ）… Email/携帯番号で検索、未登録時は招待メール送信ボタン
