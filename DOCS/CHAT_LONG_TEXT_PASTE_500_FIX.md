# 長文ペースト時のサーバーエラー（500）対策・ログ移行対応

## 事象
Chatwork 等から大量ログをコピー＆ペーストして送信すると、`POST api/messages.php` が 500 を返し「サーバーエラーが発生しました」となる。

## 原因（過去）
- 1000文字以上の長文は自動でPDFに変換しているが、**極端に長いテキスト**（数十万文字以上）を TCPDF に渡すとメモリ不足で Fatal になる。
- PDF 変換に失敗した場合、本文をそのまま DB の `content`（TEXT ≒ 64KB）に保存しようとしても収まらずエラーになる。

## 現在の対応（貼り付け上限200M・長文テキストモード）

### 貼り付け上限
- **200M（2億）文字**まで1回の送信で受け付け（`api/messages.php` の `$maxContentChars = 200000000`）。
- 200M を超える場合は「メッセージは2億文字までです。長いログは分割して送信してください。」で 400 を返す。

### 長文テキストモード（PDFにしない）
- **バイト長 > 65KB** または **文字数 > 10万** の場合は、PDF 変換を行わず「長文テキストモード」で保存する。
  - `content`: 表示用ラベル（先頭500文字＋「…（全文 X 文字は検索・AI学習用に保存されています）」、65KB 以内）。
  - `extracted_text`: 元の全文をそのまま保存（**LONGTEXT**。約4GBまで格納可能）。
- 1貼り付け＝1メッセージ（1 INSERT）。ログ移行に最適。

### DB・PHP の準備
- **extracted_text**: 200M 文字格納のため **LONGTEXT 必須**。`database/migration_messages_extracted_text_longtext.sql` を実行する（既存の FULLTEXT インデックス `ft_extracted_text` がある場合は DROP 後に MODIFY）。
- **php.ini**: 200M 文字（UTF-8 で約 200MB～600MB）の POST を受けられるようにする。目安: `post_max_size` 768M 以上、`memory_limit` 1024M 以上。

### ログ移行の推奨
- 専用グループ（例: 「Chatworkログ 2016-2026」）を作り、**テキストのまま**貼り付け。1回あたり 500万文字程度を目安に、超える場合は年度・期間で分割して貼る。
- 判断材料フォルダは要約・方針・重要抜粋向け。ログ全文は会話メッセージの `extracted_text` に任せる運用を推奨。

## 過去の対応内容（短い長文のPDF変換まわり）

### 1. PDF変換の上限（15万文字）
- **api/messages.php**: 長文送信時に「15万文字を超える場合は変換しない」判定を追加。超えた場合は 400 で「15万文字以内に分割して送信してください」と返す。
- **includes/pdf_helper.php**: `textToPdf()` の冒頭で `mb_strlen($text) > 150000` の場合は何もせず `null` を返す。TCPDF に渡す前に打ち切り、メモリ不足を防ぐ。
- 上記は**長文テキストモードでない場合**のみ適用（バイト長≤65KB かつ 文字数≤10万 の 1000字以上は従来どおり PDF 変換）。

### 2. エラーメッセージの整理
- PDF 変換で例外（メモリ不足等）が出た場合の案内を「15万文字以内に分割」に統一。
- DB の content 制限（約64KB）に引っかかった場合のメッセージを「長文はPDFに自動変換されますが、変換に失敗した場合は約6万5千文字以内に分割して送信してください」に変更。長文テキストモードの場合はラベルにしているため 65KB チェックをスキップ。

### 3. 既存の長文上限
- 送信テキスト全体の上限は**200M 文字**。長文テキストモード時は 1 貼り付け＝1 メッセージで `extracted_text`（LONGTEXT）に全文を保存。

## 関連ファイル
- `api/messages.php` … 貼り付け上限200M・長文テキストモード・PDF変換分岐・エラーメッセージ
- `includes/pdf_helper.php` … `textToPdf()` の文字数上限
- `database/migration_messages_extracted_text_longtext.sql` … extracted_text を LONGTEXT に変更
- `api/ai.php` … suggest_reply でメッセージ本文を `COALESCE(extracted_text, content)` で取得（AI学習に全文利用）
- `includes/chat/scripts.php` … 1000文字以上送信時のコンソール案内「長文はテキストのまま保存され、検索・AI学習に利用されます」

## 本番でまだ 500 が出る場合
- サーバーの PHP `error_log` で Fatal のスタックトレースを確認する。
- リクエストボディが非常に大きい（例: 数百MB）場合は、`php://input` 読み込み時点でメモリ不足になる可能性がある。`post_max_size` と `memory_limit` を 200M 文字対応の目安（上記）に上げる。
