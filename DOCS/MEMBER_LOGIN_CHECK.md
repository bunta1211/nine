# メンバーのログイン可否の確認

## ログインできる条件（コード上の条件）

- **users.email** がそのメールアドレスである
- **users.status** が `'active'` である
- **users.password_hash** が、入力したパスワード（例: clover123）と `password_verify()` で一致する

上記のすべてを満たしていればログインできます。

---

## 登録済み（承諾前）のメンバーについて

- 新規で組織に追加したメンバーは、**登録時点ではパスワードがランダム**で保存されています。
- 現状は招待メール未設定のため、**組織管理で登録し本人にパスワードを伝える運用**です。
- 招待メールを送る運用の場合は、本人が**招待メールのリンク（accept_org_invite.php?token=...）を開き、そこでパスワードを設定**すると、`users.password_hash` が更新されログインできるようになります。

つまり承諾前のままでは、**希望パスワードではログインできない**のが通常の動きです。

---

## clover.kubomomoe@gmail.com が clover123 でログインできるようにする方法

### 方法 A: 本人に招待リンクからパスワードを設定してもらう（推奨）

1. メンバー管理で該当メンバーの **「送信」** を押し、招待メールが届くようにする（SES 設定済みであればメールが届く）。
2. 届いたメールの **「パスワードを設定して承諾する」** リンクをクリックしてもらう。
3. 画面で **パスワードに clover123** を入力して送信してもらう。
4. これで `users.password_hash` が clover123 のハッシュに更新され、**clover123 でログインできる状態**になります。

### 方法 B: 管理画面でパスワードを設定する

1. **メンバー管理** で **clover.kubomomoe@gmail.com** の行の **編集（鉛筆アイコン）** を開く。
2. **パスワード** 欄に **clover123** を入力して保存する（パスワードを変更する機能がある場合）。
3. 保存後、そのメールアドレスと clover123 でログインできるか確認する。

※ 編集画面に「パスワード」欄がない場合は、方法 A で招待リンクから設定してもらう必要があります。

---

## ログインできるかどうかの確認手順

1. ブラウザで **https://social9.jp/** を開く（または index.php）。
2. **メールアドレス:** clover.kubomomoe@gmail.com  
   **パスワード:** clover123  
   でログインを試す。
3. ログインできれば「ログインできる状態」です。  
   「メールアドレスまたはパスワードが正しくありません」と出る場合は、まだ上記のどちらかの方法でパスワードが設定されていないか、status が active でない可能性があります。

---

## DB で確認したい場合（参考）

次の SQL で、該当ユーザーが存在するか・status が active か・組織の accepted_at が入っているかだけ確認できます（パスワードの中身は見えません）。

```sql
SELECT u.id, u.email, u.display_name, u.status, om.accepted_at
FROM users u
LEFT JOIN organization_members om ON om.user_id = u.id
WHERE u.email = 'clover.kubomomoe@gmail.com';
```

- **status** が `active` かつ **accepted_at** が NULL でない（日時が入っている）と、招待承諾済みです。
- 承諾済みでも「パスワードが違う」と出る場合は、まだ招待リンクでパスワードを設定していないか、別のパスワードが設定されている状態です。
