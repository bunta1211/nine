# 計画書：スワイプで3ページ移動＋縁の「光のすき間」（携帯版）

## 1. 概要

携帯版チャット画面を**3ページ（1＝中央、2＝左、3＝右）**の横並びとして扱い、**左右の収納ボタンは廃止**する。ページ間の移動は**スワイプのみ**で行い、**縁の「光のすき間」（エッジグロー）**で「どちら側にページがあるか」を表示する。あわせて**設定メニューと個人アカウントを3ページすべてに配置**する。

---

## 2. 3ページの定義と操作

| ページ | 対応パネル | 内容 |
|--------|------------|------|
| 1ページ | 中央パネル | チャット本文・ヘッダー・メッセージ入力。 |
| 2ページ | 左パネル | 会話一覧・グループ作成・友達追加・フィルタータブ。 |
| 3ページ | 右パネル | 詳細（概要・メディア・タスク・グループ設定など）。 |

- **移動操作**: スワイプのみ。ボタンは使わない。
  - 中央（1ページ）から**右方向スワイプ** → 左パネル（2ページ）へ移動。
  - 中央（1ページ）から**左方向スワイプ** → 右パネル（3ページ）へ移動。
  - 左パネルから**左方向スワイプ** → 中央へ戻る。
  - 右パネルから**右方向スワイプ** → 中央へ戻る。
- 対象は**携帯版（max-width: 768px）**。PC 版は従来どおり左・右パネルを折りたたみ／ボタンで開閉する構成を維持する。

---

## 3. 縁の「光のすき間」でページの存在を示す

**方針**: ボタンではなく、**画面（またはメインコンテナ）の左右の縁**に細いグラデーションを常時表示し、「その方向にページがある」ことを知らせる。

| 表示する縁 | 意味 |
|------------|------|
| **左縁の光** | 左側にページ（2ページ＝左パネル）がある。右方向スワイプで移動できる。 |
| **右縁の光** | 右側にページ（3ページ＝右パネル）がある。左方向スワイプで移動できる。 |

- **現在のページに応じた出し分け（案）**
  - 中央（1ページ）: **左縁と右縁の両方**に光のすき間を表示。
  - 左パネル（2ページ）: **右縁のみ**に光のすき間（中央・右へ戻れることを示す）。左縁は表示しないか弱める。
  - 右パネル（3ページ）: **左縁のみ**に光のすき間（中央・左へ戻れることを示す）。右縁は表示しないか弱める。

- **実装イメージ**
  - メインコンテナ（例: `.main-container` または body 直下のラッパー）に、`::before`（左縁用）と `::after`（右縁用）で幅 8〜12px 程度の `linear-gradient` を重ねる。pointer-events: none でタッチは透過。
  - または、固定位置の左右 2 本の細い div を置き、現在ページに応じて opacity や display を切り替える。
  - 色は白→透明、またはテーマのアクセントを薄めたもの→透明。既存の [includes/design_config.php](includes/design_config.php) のトークンに合わせる。

---

## 4. ボタン廃止の内容

- **携帯版で非表示・削除するもの**
  - 上パネル内の**左収納ボタン**（.toggle-left-btn）: 携帯では表示しない。
  - 上パネル内の**右収納ボタン**（.toggle-right-btn）: 携帯では表示しない。
  - JS で動的追加している**左用の ⇒ ボタン**（.mobile-menu-btn）: 追加しない、または削除する。
- **HTML は残し、CSS で非表示**にするか、**携帯用に topbar から該当ボタンを出さない**ようにする。PC 版では従来どおり表示する。

---

## 5. スワイプで3ページに移動する仕様

- **レイアウト案**
  - 携帯では、メインコンテナ内に**横並び3カラム**（左パネル | 中央パネル | 右パネル）を並べ、親要素で `overflow-x: hidden` と `scroll-snap-type: x mandatory`（または JS でスナップ）を指定。表示幅は 1 ページ分（100vw）とし、スワイプで横スクロール（または transform: translateX）してどのページを見せるか切り替える。
  - 既存の「オーバーレイ＋スライドイン」ではなく、**3つが常に横に並んだ1本のストリップ**として扱う方式に変更する。
- **スワイプの検知**
  - 既存の [assets/js/chat-mobile.js](assets/js/chat-mobile.js) の touch 系（touchstart, touchmove, touchend）を拡張するか、3ページ用のスワイプハンドラを新設。閾値（例: 50px）を超えたらページを切り替え、スナップ位置（0%, 100%, 200% など）にアニメーションで移動。
- **初期表示**
  - 常に中央（1ページ）を表示。左・右パネルは DOM 上は存在するが、ビューポート外（左側・右側）に配置されている状態から開始。

---

## 6. 設定メニュー・個人アカウントの配置

**方針**: 1・2・3ページのいずれを表示しているときも、**設定**と**個人アカウント**にアクセスできるようにする。

- **1ページ（中央）**: 現状どおり上パネル（.top-panel）に設定リンク＋個人アカウントを配置。
- **2ページ（左パネル）**: 左パネルのヘッダー（.left-header）に、設定リンク（歯車）＋個人アカウント（ドロップダウン）を追加。
- **3ページ（右パネル）**: 右パネルのヘッダー（.right-header）に、設定リンク＋個人アカウントを追加。

上パネルは「常に画面上部に固定」するか、あるいは「現在表示中のページに属するヘッダーだけ表示する」かは実装で決める。3ページを横ストリップで扱う場合、**各ページごとにヘッダー（ロゴ・設定・アカウント）を持たせ、スワイプでそのページのヘッダーが一緒に動く**形にすると、どのページにいるかが分かりやすい。

## 9. 注意事項・依存

- **PC 版**: 本計画の対象は携帯版のみ。PC では従来どおり左・右の収納ボタンでパネルを開閉する。
- **既存のオーバーレイ・閉じるボタン**: 左パネル・右パネル内の「×」閉じるボタンは、スワイプで中央に戻れるため、残すか「スワイプで戻る」案内に変えるかは検討事項。
- **変数**: 左・右パネルにアカウントドロップダウンを置く場合、$user, $display_name 等が chat.php から include 先に渡っているか確認する。
- **デザイン規格**: [DOCS/STANDARD_DESIGN_SPEC.md](DOCS/STANDARD_DESIGN_SPEC.md) および [includes/design_config.php](includes/design_config.php) に合わせる。

- 画面の左縁・右縁の「光のすき間」で、その方向にページがあることが分かること（現在ページに応じて表示する縁を切り替える）。
