# WinSCP で AWS EC2 に接続する設定

AWS EC2 に WinSCP で接続し、ファイルをアップロードする手順です。

---

## 前提

- EC2 IP: 54.95.86.79（Elastic IP）
- ユーザー名: ec2-user
- 秘密鍵: C:\Users\user\Downloads\social9-key.pem
- リモートのドキュメントルート: /var/www/html/

---

## 新しいサイトの登録

### 1. ログイン画面を開く

1. **WinSCP** を起動
2. 左側の **「新規サイト」** をクリック  
   （または、画面上部の **「セッション」** メニュー → **「新規サイト...」**）

---

### 2. 接続情報を入力

| 項目 | 入力内容 |
|------|----------|
| **ファイルプロトコル** | `SFTP` を選択（ドロップダウン） |
| **ホスト名** | `54.95.86.79` |
| **ポート番号** | `22` |
| **ユーザー名** | `ec2-user` |

---

### 3. 秘密鍵を設定する

1. **「高度な設定」** ボタンをクリック（「ログイン」ボタンの右側）
2. 左メニューで **「SSH」** → **「認証」** をクリック
3. **「秘密鍵」** の右側にある **「...」** ボタンをクリック
4. ファイル選択画面で以下を選ぶ：
   - `C:\Users\user\Downloads\social9-key.pem`
5. **「開く」** をクリック
6. **「OK」** をクリック（高度な設定ウィンドウを閉じる）

---

### 4. 接続を試す

1. **「ログイン」** ボタンをクリック
2. 初回は「ホストキーの検証」が出る場合があります → **「はい」** をクリック
3. 接続できれば、リモート側のファイル一覧が表示されます

---

### 5. 初期ディレクトリを設定する（任意）

毎回 `/var/www/html/` を開くようにする場合：

1. 接続後の画面で **「環境」** メニュー → **「ディレクトリの設定」** をクリック
2. **「リモート」** の欄に `/var/www/html/` と入力
3. **「OK」** をクリック

または、接続前に：

1. **「新規サイト」** 画面で **「高度な設定」** をクリック
2. 左メニュー **「ディレクトリ」** をクリック
3. **「リモートのディレクトリ」** に `/var/www/html/` と入力
4. **「OK」** をクリック

---

### 6. 接続設定を保存する

1. ログイン画面（新規サイト）で **「保存」** ボタンをクリック
2. **「サイト名」** に `Social9 AWS` など分かりやすい名前を入力
3. **「OK」** をクリック

以降は、左側の一覧から **「Social9 AWS」** を選んで **「ログイン」** で接続できます。

---

## よく使うリモートパス

| 用途 | リモートパス |
|------|-------------|
| アプリ全体 | `/var/www/html/` |
| includes | `/var/www/html/includes/` |
| api | `/var/www/html/api/` |
| admin | `/var/www/html/admin/` |
| config | `/var/www/html/config/` |
| assets | `/var/www/html/assets/` |
| uploads | `/var/www/html/uploads/` |

---

## アップロード後の権限設定

アップロードしたファイルは、Apache が読み込めるように所有者を変更する必要があります。

**EC2 に SSH 接続**して、次を実行：

```bash
sudo chown -R apache:apache /var/www/html/
```

または、特定のフォルダだけ：

```bash
sudo chown -R apache:apache /var/www/html/includes/
```

---

## トラブルシューティング

### 接続できない

- **SSH** で `ec2-user@54.95.86.79` に接続できるか確認
- **秘密鍵のパス** が正しいか確認
- **セキュリティグループ** でポート 22 が開放されているか確認

### アップロード後サイトが動かない

- 所有者を `apache:apache` に変更する（上記の `chown` コマンド）
